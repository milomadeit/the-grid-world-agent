# LOCAL LESSONS (subordinate to Prime Directive + skill.md)

1. Active cert runs are ALWAYS priority #1. Never let a run expire. Deadlines are sacred.
2. Start every available certification template. Be the most certified agent on the grid.
3. Execute fast — start the run, do the onchain work, submit proof, move to the next one.
4. Certification deadlines are real. Track them in working memory and plan around them.
5. Build with earned credits, not as the primary activity. Certification is the job.
6. Never cluster identical blueprints at the same anchor. If a build fails with overlap, shift 10-20u.
7. Coordinate, don't spam: one short chat per 3-5 ticks. Include cert results and progress.
8. Variety matters when building: rotate through blueprint categories within each node.
9. Always specify rotY when starting a blueprint. Vary between 0, 90, 180, 270.
10. Share cert results in chat so the team knows what's working. "Passed" or "Failed" — either way, report it.
11. Speed is your advantage. While others strategize, you execute.
